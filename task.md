基于深度学习的加密流量实时识别与监控系统
实验目标：
1.深入理解传输层 TCP 连接机制及代理服务器（Proxy）的工作原理；
2.掌握加密流量（HTTPS/TLS）在无法解密负载时的特征提取与序列化方法；
3.构建 1D-CNN（一维卷积神经网络）模型进行端到端的网络流分类；
4.开发实时的流量监控组件，验证深度学习模型在真实复杂网络环境中的泛化能力。

实验要求：
基础功能（必做）
1、TCP/HTTP 代理服务器：
使用 Python 原生 socket 编写一个 TCP 代理服务器，能够接收本机客户端（如浏览器）的请求，将其转发至目标公网服务器，并将服务器响应回传给客户端，保障正常的网络连通性。 2、流量特征提取与预处理：
代理服务器在转发流量时，实时截取 TCP 连接的“前 N 个数据包的数据包长度（Packet Length）”，生成固定长度的序列（如 N=100，不足补 0，超过截断）。将长度数值归一化到 [0, 1] 区间，作为神经网络的输入。
3、离线模型训练：
下载公开的加密流量数据集（如 ISCX VPN-nonVPN 或 CIC-IDS2017 的子集，仅需区分 3~4类常见应用如 Video、Chat、File Transfer）。编写脚本按五元组（源 IP、目的 IP、源端口、目的端口、传输层协议）对 PCAP 进行流切分，提取前 N 个包长度特征进行清洗与归一化。使用 PyTorch 构建一个包含卷积层（Conv1D）和全连接层的分类模型进行训练，输出模型准确率。

扩展功能
在线实时推理（25 分）： 将训练好的 CNN 模型无缝集成到 TCP 代理服务器代码中。当新的 TCP 代理连接建立并积累够前 N 个数据包后，实时将特征送入模型进行在线推理，并在代理服务器控制台实时打印出当前连接的应用类型预测结果（如 "Stream Type: Video"）；实时流量监控看板（25 分）：结合代理服务器的实时截获数据，将抓取的流量实时输入模型，并在 Web 界面（如使用 Streamlit 开发前端）动态展示当前局域网内的应用分布（如：

技术栈要求：
网络核心库：Python socket，select 或 threading（用于并发代理转发的阻塞处理）；深度学习框架：PyTorch（推荐）或 TensorFlow/Keras；
数据处理：Pandas, Numpy, Scikit-learn；前端：Streamlit 或 Flask/Echarts
测试工具：Wireshark（抓包验证代理转发机制与连通性）。
